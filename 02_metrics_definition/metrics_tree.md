## 지표 분해 공식

### GMV 분해
```
GMV = 송금 GMV + QR 결제 GMV

송금 GMV = 송금 사용자 수 × 인당 송금 건수 × 건당 평균 송금액
         = (DAU × 송금 사용률) × 인당 건수 × 평균액

QR 결제 GMV = QR 결제 사용자 수 × 인당 QR 건수 × 건당 평균 결제액
```

### DAU 분해
```
DAU = 신규 사용자 + 복귀 사용자

신규 사용자 = 신규 설치 × 가입 전환율 × 첫 로그인율
복귀 사용자 = 기존 사용자 × 리텐션율
```

### Revenue 분해
```
Revenue = 송금 수수료 + QR 결제 수수료 + 금융상품 수수료

송금 수수료 = 유료 송금 건수 × 건당 수수료
           = (송금 사용자 × 유료 비율 × 건수) × 수수료
```

## 문제 진단 시나리오

### 시나리오 1: DAU 하락

```
DAU 하락 (↓10%)
  ├── 신규 사용자 감소?
  │     ├── 설치 수 감소? → 마케팅 효율 확인
  │     └── 가입 전환율 하락? → 가입 퍼널 점검
  └── 복귀 사용자 감소?
        ├── D1 리텐션 하락? → 온보딩 개선
        ├── D7 리텐션 하락? → 핵심 기능 사용성
        └── D30 리텐션 하락? → 서비스 가치 재검토
```

### 시나리오 2: GMV 하락

```
GMV 하락 (↓15%)
  ├── 송금 GMV 하락?
  │     ├── 송금 사용자 수 감소? → DAU 트리 확인
  │     ├── 인당 건수 감소? → 서비스 불편 확인
  │     └── 건당 금액 감소? → 사용 패턴 변화
  └── QR 결제 GMV 하락?
        ├── QR 결제 사용자 감소? → QR 노출 확인
        └── 건당 금액 감소? → 가맹점 구성 변화
```
